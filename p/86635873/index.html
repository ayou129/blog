简述

Git 是一种非常有用的版本控制工具，可以让用户轻松地跟踪文件的更改历史，并与其他用户协作开发项目。

对比

软件 介绍 分支 其他
Git 分布式(每个本地仓库都是一个完整的备份，可以独立运作，不需要中央服务器) 分支功能强大 性能更快一些
SVN 集中式(需要中央服务器才能工作) 分支功能较弱 -

工作流程

在本地主机上创建 Git 仓库。
在 Git 仓库中编辑文件，并保存更改。
提交更改到 Git 仓库，以便将它们记录到版本历史中。
通过远程仓库与其他用户协作，以便快速完成项目。

在整个工作流程中，Git 还提供了许多有用的功能，如分支管理、标签和撤销更改等。

区域

跟踪文件 暂存区 仓库
(已修改 未提交 已修改 已提交)

未跟踪–>add后跟踪文件，变为未修改，等待commit–>commit后 暂存区(进入暂存区同时将状态更换为未修改)
修改文件后，变为已修改，等待commit–>commit后 暂存区(进入暂存区同时将状态更换为未修改)
touch file add后 进入缓存区 等待commit
修改文件后 add后 进入缓存区 等待commit

与源服务器交互

  • git push origin master 上传到远程服务器
  • git status 查看当前版本库状态
  • git add 跟踪某文件
  • git add . 跟踪所有文件
  • git commit -a -m ‘message’ 添加文件到暂存区（-a 将已跟踪的文件直接放入仓库中）
  • git commit –amend 撤销commit并重新提交
  • git rm 从git删除某文件记录以及文件
  • git rm –cached 从git删除某文件记录
  • git rm -r –cached 从git删除某文件记录（备用）
  • git mv 将git中的某个文件重命名

查看工作区

  • git diff 查看 当前工作区 与 暂存区 的不同处
  • git diff –cached 查看 暂存区 与 HEAD(上次提交时)的不同处
  • git diff –staged 同上

分支相关

  • git branch -a 查看当前所有分支
  • git branch ask 新建分支
  • git checkout ask 切换分支
  • git checkout -b ask 新建并切换到该分支
  • git branch -d ask 删除分支
  • git branch -D ask 强制删除分支
  • git merge ask 合并分支
  • git branch –merged 查看合并的分支
  • git branch –no-merged 查看未合并的分支

缓存相关

  • git stash 添加缓存
  • git stash list 查看缓存列表
  • git stash apply stash@{0}恢复缓存不删除缓存记录 注意切换分支
  • git stash pop stash@{0} 恢复缓存并且删除缓存记录
  • git stash drop stash@{0} 删除缓存
  • git log –name-only

git库所在的文件夹中的文件大致有4种状态

Untracked:

未跟踪, 此文件在文件夹中, 但并没有加入到git库, 不参与版本控制. 通过git add 状态变为Staged.

Unmodify:

文件已经入库, 未修改, 即版本库中的文件快照内容与文件夹中完全一致. 这种类型的文件有两种去处, 如果它被修改,
而变为Modified. 如果使用git rm移出版本库, 则成为Untracked文件

Modified:

文件已修改, 仅仅是修改, 并没有进行其他的操作. 这个文件也有两个去处, 通过git add可进入暂存staged状态,
使用git checkout 则丢弃修改过, 返回到unmodify状态, 这个git checkout即从库中取出文件, 覆盖当前修改

Staged:

暂存状态. 执行git commit则将修改同步到库中, 这时库中的文件和本地文件又变为一致, 文件为Unmodify状态.
执行git reset HEAD filename取消暂存, 文件状态为Modified

Git 状态 untracked 和 not staged的区别
1）untrack 表示是新文件，没有被add过，是为跟踪的意思。
2）not staged 表示add过的文件，即跟踪文件，再次修改没有add，就是没有暂存的意思
