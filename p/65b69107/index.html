Nginx

概念

  1. (正向)代理：客户端请来的代理
  2. 反向代理：服务器请来的代理
  3. 集群
  4. 高可用：软件 shutdown 之后可以继续使用，一个系统经过专门的设计，从而减少停工时间，而保持其服务的高度可用性
  5. 高并发

nginx.conf

location

匹配方式 + 执行顺序

如果在匹配中没有直接返回，则会一直继续匹配
精准匹配不会被覆盖，一般匹配会被覆盖

  1. 精准匹配
    • location 无前缀
    • location =
  2. 一般匹配，但先匹配最长的
    • location ^
    • location ^~ 不使用正则
  3. 正则匹配
    • location ~ 区分大小写，严格模式
    • location ~* 正则不区分大小写
    • location ~* .(gif|png|css|js)$

Nginx新建站点

  1. vim /usr/local/nginx/conf/nginx.conf 在http的最下方include一个站点文件 include default_server.conf;
  2. vim default_server.conf 新建站点文件
  3. /usr/local/nginx/sbin/nginx -s reload 重新加载配置文件，服务不会中断。而且reload时会测试conf语法等，如果出错会rollback用上一次正确配置文件保持正常运行。

遇到的问题

nginx: [error] invalid PID number "" in "/usr/local/nginx/logs/nginx.pid"
解决方案：

  1. /usr/local/nginx/sbin/nginx -c /usr/local/nginx/conf/nginx.conf 指定文件加载nginx
  2. /usr/local/nginx/sbin/nginx -s reload
