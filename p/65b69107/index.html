Nginx

创建于：2016年4月25日上次更新：2022年9月10日

简述

一个高性能的HTTP和反向代理web服务器。

作用：

  • 提供静态内容：nginx 可以用来托管网站的静态内容（如 HTML、CSS、JavaScript 等文件），并通过 HTTP 协议将这些内容返回给用户浏览器。
  • 提供动态内容：nginx 可以通过反向代理的方式与后端的 Web 应用程序集成，并将用户请求转发给后端应用程序，然后将应用程序的响应返回给用户浏览器。
  • 处理 SSL/TLS 加密：nginx 可以处理 SSL/TLS 加密，使网站能够通过 HTTPS 协议与用户浏览器通信，并保护用户隐私和数据安全。
  • 实现负载均衡：nginx 可以通过负载均衡功能将用户请求平摊到多台后端服务器，提高网站的可用性和性能。
  • 提供反向代理和虚拟主机：nginx 可以通过反向代理和虚拟主机功能，使多个网站能够共享同一个 IP 地址和端口，并通过不同的域名访问各自的网站。

名词解释：

  • IO复用 同时等待多个文件描述符就绪，以系统调用的形式提供，常见的技术有：select、poll、epoll、kqueue等待。
  • epoll (Linux下实现事件队列的方式)Linux内核为处理大批量文件描述符而作了改进的poll，是Linux下多路复用IO接口select/poll的增强版本，它能显著提高程序在大量并发连接中只有少量活跃的情况下的系统CPU利用率。
  • kqueue (是UNIX(MacOS)下实现事件队列的方式)
  • iocp (Windows下实现事件队列的方式)

工作原理

nginx 采用事件驱动的架构，使用一个主进程和若干子进程组成(1：n)。

  • 主进程 启动时，会创建若干个 子进程 ，并监听指定的网络端口。
  • 当有网络请求到达时，主进程 会接收该请求并分发给某个 子进程 处理。
  • 子进程 会解析请求报文并提取有效信息，根据配置的规则处理请求并生成响应，静态动态是否传递给后端应用程序等。
  • 当处理完成后，子进程 会将响应返回给主进程，主进程 再将响应发送给客户端。

匹配方式 + 执行顺序

如果在匹配中没有直接返回，则会一直继续匹配
精准匹配不会被覆盖，一般匹配会被覆盖

    • location 无前缀
    • location =
  2. 一般匹配，但先匹配最长的
    • location ^
    • location ^~ 不使用正则
  3. 正则匹配
    • location ~ 区分大小写，严格模式
    • location ~* 正则不区分大小写
    • location ~* .(gif|png|css|js)$
